把翻译放进真正的阅读现场

LingFlow 不要求你离开当前页面、应用或设备。它把翻译入口放到阅读发生的地方，再把服务配置统一收拢起来。

网页阅读

双语内容直接进入页面

浏览器扩展会抽取可读正文段落，将译文注入到原页面中，并支持清理和去重，适合阅读长文、技术文档和外文资料。

桌面划词

选中文本即可唤起浮窗

Windows 客户端支持全局划词悬浮窗，默认通过 UI Automation 读取选区，不主动模拟复制，尽量减少对剪贴板的打扰。

移动端

Android 工作台复用同一核心

Tauri Android 工程已接入同一套 React 控制台和翻译核心，适合移动端输入、复制译文、查看历史和管理服务源。

多引擎翻译 本地优先管理

你可以接入常用云翻译，也可以接入兼容 OpenAI 的 AI 服务。密钥、配置和历史由本地客户端管理，浏览器扩展默认复用桌面端能力。

从扩展到桌面端的本地通道

浏览器扩展默认连接桌面端代理，读取当前服务源配置，并由桌面端转发翻译请求。网页选区也可以上报到桌面端，触发同一个划词浮窗。
