核心能力
翻译不应打断思考的流动
灵流围绕"无感翻译"设计每一个交互——从网页双语注入到全局划词悬浮窗，翻译自然地融入工作流，而非跳出来抢占注意力。

网页双语注入

译文以略小一号、略淡一级的颜色自然嵌入原文下方，保持页面原貌。鼠标悬停高亮当前段落，一键清除所有注入内容。支持视口懒加载翻译，不浪费 API 额度。

全局划词翻译

Windows 桌面端支持任意应用内选中文本即弹出翻译。使用 UI Automation 读取选区，不模拟 Ctrl+C、不污染剪贴板。浏览器中由扩展辅助上报选区，体验一致。

AI 多服务源与自动回退

配置多个 OpenAI-compatible 服务源，每个服务源可填多个模型。开启自动回退后，翻译失败会依次尝试当前源的下一个模型，再尝试下一个服务源——确保翻译永不中断。

密钥完全本地化

采用 BYO-Key 模式——所有 API Key 保存在你的设备上。桌面端使用系统凭据库（Windows Credential Manager），扩展端通过桌面端本地代理转发请求，密钥从不离开本机。

桌面端 + 扩展联动

浏览器扩展通过本地代理 (127.0.0.1:47631) 读取桌面端配置并转发翻译请求。在桌面端配置一次服务源，扩展自动同步——无需重复填写密钥。

在线更新

桌面端自动检查 GitHub Releases 最新版本，发现新版本后引导下载。后续将支持静默下载和重启安装，让更新无感。

全平台覆盖
一套核心代码，多端运行
基于 pnpm workspace + Turborepo 的 Monorepo 架构，共享 @lingflow/core 翻译核心和 @lingflow/dom 注入引擎。React + Tailwind CSS 驱动所有终端 UI。

隐私与安全
你的密钥，只在你手中
灵流的架构设计从第一天就遵循一个原则：用户的 API Key 绝不离开本地设备。

端到端本地存储

桌面端密钥保存在 Windows Credential Manager 系统凭据库中，非机密数据存入本地 SQLite。扩展通过本机 localhost 代理读取桌面端配置，所有翻译请求由桌面端 Rust HTTP 层代理转发。没有中心服务器、没有云端存储、没有遥测上报。

快速开始
三分钟上手灵流
01

安装桌面端

从 GitHub Releases 下载最新 Windows 安装包。需要 WebView2 Runtime（Windows 11 已内置）。

02

配置翻译服务源

在桌面端配置至少一个 Provider 的 API Key。AI 翻译用户可添加多个 OpenAI-compatible 服务源并启用自动回退。

03

安装浏览器扩展

构建扩展后从 apps/extension/dist 加载到 Chrome/Edge。扩展自动连接桌面端代理，无需重复配置密钥。

04

开始双语阅读

打开任意网页，点击扩展图标开启整页翻译，或选中文本查看翻译。桌面端开启全局划词后，任意应用内选中文本即可翻译。
